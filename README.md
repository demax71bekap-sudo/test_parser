# ParserApp — Универсальный парсер прайс-листов

Автоматизированный сбор данных с сайтов формата `https://XXX.nt-rt.ru/price`

## 🚀 Быстрый старт

### 1. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Запуск (режим разработки)
```bash
python main.py
```

### 3. Сборка в EXE
```bash
pip install pyinstaller
pyinstaller pyinstaller.spec
```
Готовый файл появится в папке `dist/ParserApp.exe`

## 📁 Структура конфигов

### config/sources.json
Добавление нового бренда:
```json
{
  "newbrand": {
    "name": "New Brand",
    "price_url": "https://newbrand.nt-rt.ru/price",
    "manufacturer_url": "https://newbrand.com",
    "selectors_profile": "nt_rt_default"
  }
}
```

## 📊 Формат экспорта Excel

| Колонка | Описание |
|---------|----------|
| Артикул | Артикул товара |
| Название | Полное наименование |
| Цена | Цена в рублях |
| Категория | Категория из прайса |
| Описание | Очищенное описание (этап 2) |
| Изображение | Путь к файлу (этап 2) |
| URL | Ссылка на товар |
| [Характеристики] | Динамические колонки (этап 2) |

## ⚙️ Настройки

### config/settings.json
- `http.delay_min/max` — задержка между запросами (сек)
- `http.max_retries` — количество повторных попыток
- `http.user_agents` — пул User-Agent для ротации

## ⚠️ Важно

1. Соблюдайте задержки между запросами (настроено в settings.json)
2. Уважайте robots.txt целевых сайтов
3. Не превышайте разумную частоту запросов
4. Используйте только в законных целях

## 📞 Поддержка

- Логи сохраняются в папке `logs/`
- Ошибки парсинга — `logs/errors.log`
- Основной лог — `logs/parser.log`
